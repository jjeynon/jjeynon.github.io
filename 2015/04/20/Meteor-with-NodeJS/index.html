Meteor with NodeJS

Create a user called “meteor” with no password that must have sudo rights to work

$ useradd meteor

Install Meteor as the meteor user

$ sudo -u mclaren curl https://install.meteor.com/ | sh

Create directory

$ mkdir /opt/mclaren

Change ownership of the directory for group meteor

$ sudo chown meteor:meteor /opt/mclaren

Change into the mclaren directory

$ cd /opt/mclaren

Pull the threesixty app from the git repository

$ sudo -u meteor git clone https://jjeynon@bitbucket.org/johnsanders/threesixty.git

Change directories into /opt/mclaren/ and check the permissions

$ ls -la

drwxr-xr-x. 7 meteor meteor 4096 date time threesixty

Change directories into /opt/mclaren/threesixty

$ cd /opt/mclaren/threesixty

#Start the application as the meteor user

$ sudo -u meteor /usr/local/bin/meteor

#Push code changes back to Bit Bucket with GIT

Check the file changes to be synced with GIT

$ git status

And if needed check the difference $ git diff

Add the files to upload

$ sudo git add .

Commit the file changes to Bit Bucket

$ git commit –m “commit message here ”

Push the origin to the master

$ sudo git push origin master

A final check to confirm the code file changes are synced

$ git status

#To kill the process running Meteor

Check the PID values of the process running

$ ps aux | grep meteor

kill the process with the PID number

$ sudo kill -9 [pid number]

#Remove Meteor and the application directory from the host machine

Go to the /opt/mclaren directory

$ sudo rm –rf threesixty

$ ls –la

Then proceed to re-clone the threesxity application back to /opt/mclaren

$ sudo -u meteor git clone https://jjeynon@bitbucket.org/johnsanders/threesixty.git

Check the files and permissions are correct /opt/mclaren/threesixty

$ ls -la

#Future Work

If Mongo is installed as a stand-alone DB, then Mongo backup and restores can be performed.

$ mongodump -h 127.0.0.1:3002 –db meteor –out ../backups

$ mongorestore -h 127.0.0.1:3001 –db meteor ../cleandb_elex2015/
